# KarvisForAll — 部署指南

## 部署方式总览

| 方式 | 适合场景 | 复杂度 |
|------|----------|--------|
| [一键脚本 `setup.sh`](#方式一一键脚本) | 本地试用 / 快速体验 | ⭐ |
| [Docker Compose](#方式二docker-compose) | 服务器长期运行（推荐） | ⭐⭐ |
| [手动部署](#方式三手动部署) | 不想用 Docker / 自定义环境 | ⭐⭐ |
| [Git Push 云服务器](#方式四git-push-云服务器部署) | 开发者持续更新代码 | ⭐⭐⭐ |

## 前置准备

### 必备三件套

| 项目 | 说明 | 获取方式 |
|------|------|----------|
| **DeepSeek API Key** | AI 大脑 | https://platform.deepseek.com/ 注册，充值 10 元 |
| **企微应用** | 消息通道 | https://work.weixin.qq.com/ 注册企业，创建应用 |
| **服务器/电脑** | 运行环境 | 腾讯云轻量 1C1G（¥30-60/月）或自己电脑 |

### 企微应用需要记录的信息

| 信息 | 在哪找 |
|------|--------|
| Corp ID（企业 ID） | 企业信息页 |
| AgentId | 应用详情页 |
| Secret | 应用详情页 |
| Token + EncodingAESKey | 应用 → 接收消息 → 设置 API 接收（点随机生成） |

> "接收消息"的 URL 先不填，等 Karvis 启动后再配。

## 方式一：一键脚本

```bash
cd KarvisForAll
./setup.sh
```

脚本自动完成：
- 检查 Python 3.9+
- 安装依赖
- 引导填写 DeepSeek API Key、企微配置
- 自动生成 ADMIN_TOKEN
- 安装 cloudflared 内网穿透
- 启动服务 + 生成公网 URL

启动成功后：

```
╔══════════════════════════════════════════════════════════════╗
║  公网 URL 已生成!                                            ║
╚══════════════════════════════════════════════════════════════╝
  你的公网地址: https://xxx-xxx-xxx.trycloudflare.com
  企微后台 URL 填: https://xxx-xxx-xxx.trycloudflare.com/wework
```

> ⚠️ cloudflared 地址每次重启会变。需要稳定地址请用服务器部署。

## 方式二：Docker Compose

### 安装 Docker

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com | sh
sudo systemctl start docker && sudo systemctl enable docker
```

### 部署步骤

```bash
# 1. 上传代码到服务器并进入目录
cd KarvisForAll

# 2. 配置环境变量
cp .env.example src/.env
nano src/.env    # 填入真实配置

# 3. 启动
cd deploy
docker-compose up -d

# 4. 查看日志
docker logs -f karvis
```

### `.env` 最小配置

```bash
DEEPSEEK_API_KEY=sk-你的key
WEWORK_CORP_ID=你的企业ID
WEWORK_AGENT_ID=你的AgentID
WEWORK_CORP_SECRET=你的Secret
WEWORK_TOKEN=你的Token
WEWORK_ENCODING_AES_KEY=你的AESKey
DEFAULT_USER_ID=你的企微用户ID
ADMIN_TOKEN=随便写一个长密码
```

### Docker 常用命令

| 命令 | 说明 |
|------|------|
| `docker-compose up -d` | 启动（后台） |
| `docker-compose down` | 停止 |
| `docker-compose restart` | 重启 |
| `docker logs -f karvis` | 实时日志 |
| `docker logs karvis --tail 50` | 最近 50 行 |

### Docker 架构说明

```
deploy/
├── Dockerfile           # Python 3.11-slim, 端口 9000
│                        # 健康检查: curl /web/login
└── docker-compose.yml   # 容器名 karvis, 端口映射 9000:9000
                         # 数据卷 karvis_data 持久化 /app/data
```

- 数据卷 `karvis_data` 独立于容器生命周期，`docker-compose down` 不会丢数据
- 只有 `docker volume rm karvis_data` 才会删除数据

## 方式三：手动部署

```bash
# 1. 安装 Python
sudo apt update && sudo apt install python3 python3-pip -y

# 2. 进入代码目录
cd KarvisForAll/src

# 3. 安装依赖
pip3 install -r requirements.txt

# 4. 配置
cp ../.env.example .env
nano .env

# 5. 启动
python3 app.py
```

### 后台运行方式

**方式 A：nohup（简单）**

```bash
nohup python3 app.py > karvis.log 2>&1 &
tail -f karvis.log      # 查看日志
```

**方式 B：systemd（推荐，开机自启）**

```bash
sudo tee /etc/systemd/system/karvis.service << 'EOF'
[Unit]
Description=KarvisForAll AI Assistant
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/KarvisForAll/src
ExecStart=/usr/bin/python3 app.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable karvis
sudo systemctl start karvis
```

| 命令 | 说明 |
|------|------|
| `sudo systemctl status karvis` | 查看状态 |
| `sudo systemctl restart karvis` | 重启 |
| `journalctl -u karvis -f` | 实时日志 |
| `journalctl -u karvis --since "1 hour ago"` | 最近 1 小时日志 |

## 方式四：Git Push 云服务器部署

适合开发者持续迭代代码。一条命令完成部署。

### 服务器端配置（一次性）

```bash
# 1. 创建 bare repo
mkdir -p ~/karvis-repo.git && cd ~/karvis-repo.git
git init --bare

# 2. 创建 post-receive 钩子
cat > hooks/post-receive << 'HOOK'
#!/bin/bash
echo ">>> 开始部署 KarvisForAll..."
WORK_DIR=/root/KarvisForAll

# 拉取最新代码
git --work-tree=$WORK_DIR --git-dir=/root/karvis-repo.git checkout -f

# 安装依赖
cd $WORK_DIR/src
pip3 install -r requirements.txt --quiet

# 重启服务
PID=$(pgrep -f "python3 app.py" | head -1)
if [ -n "$PID" ]; then
    kill $PID
    sleep 2
fi
cd $WORK_DIR/src
nohup python3 app.py > /root/karvis.log 2>&1 &
NEW_PID=$!
echo ">>> 部署完成！PID: $NEW_PID"
HOOK

chmod +x hooks/post-receive
```

### 本地配置（一次性）

```bash
cd KarvisForAll
git remote add deploy root@你的服务器IP:~/karvis-repo.git
```

### 日常部署

```bash
git add . && git commit -m "描述"
git push deploy master
```

服务器自动执行：拉代码 → 装依赖 → 重启进程。

### 当前实际配置


## 连接企微

Karvis 启动后，去企微管理后台配置消息接收：

### 有公网 IP（服务器）

1. 应用 → 接收消息 → 设置 API 接收
2. URL 填：`http://你的IP:9000/wework`
3. Token / AESKey 填 `.env` 中的值
4. **必须**配置企业可信 IP：应用详情 → 企业可信 IP → 填服务器公网 IP

### 无公网 IP（本地电脑）

```bash
cloudflared tunnel --url http://localhost:9000
# 输出：https://abc-def-ghi.trycloudflare.com
```

企微后台 URL 填：`https://abc-def-ghi.trycloudflare.com/wework`

### 验证连接

在企微 app 中找到应用，发一条"你好"。收到欢迎消息 = 成功。

**没收到回复排查顺序**：
1. 日志中搜 `[handle_message]` — 没有说明企微消息没送达（检查 URL/防火墙 9000 端口）
2. 搜 `[Brain]` — 有说明处理出错（检查 API Key/余额）
3. 搜 `reply_text` — 有但企微没收到（检查企业可信 IP）

## 常用运维操作

### 查看日志

```bash
# Docker
docker logs -f karvis
docker logs karvis --tail 100

# systemd
journalctl -u karvis -f

# nohup
tail -f karvis.log
```

### 重启服务

```bash
# Docker
docker-compose restart

# systemd
sudo systemctl restart karvis

# 手动
kill $(pgrep -f "python3 app.py") && cd /root/KarvisForAll/src && nohup python3 app.py > /root/karvis.log 2>&1 &
```

### 数据备份

```bash
# 所有用户数据在 data/ 目录
tar czf karvis-backup-$(date +%Y%m%d).tar.gz data/

# 恢复
tar xzf karvis-backup-20260218.tar.gz
```

### 手动触发定时任务

```bash
# 触发晨报（所有活跃用户）
curl -X POST http://localhost:9000/system -H "Content-Type: application/json" -d '{"action":"morning_report"}'

# 触发日报
curl -X POST http://localhost:9000/system -H "Content-Type: application/json" -d '{"action":"daily_report"}'

# 刷新缓存
curl -X POST http://localhost:9000/system -H "Content-Type: application/json" -d '{"action":"refresh_cache"}'
```

### 检查模块加载

```bash
cd KarvisForAll/src
python3 -c "from skill_loader import load_skill_registry; r=load_skill_registry(); print(f'Skills: {len(r)}')"
# 预期输出: Skills: 37
```

## 环境变量完整清单

### 必填

| 变量 | 说明 |
|------|------|
| `DEEPSEEK_API_KEY` | DeepSeek API 密钥 |
| `DEEPSEEK_BASE_URL` | API 地址（默认 `https://api.deepseek.com/v1`） |
| `WEWORK_CORP_ID` | 企微企业 ID |
| `WEWORK_AGENT_ID` | 应用 AgentID |
| `WEWORK_CORP_SECRET` | 应用 Secret |
| `WEWORK_TOKEN` | 回调 Token |
| `WEWORK_ENCODING_AES_KEY` | 回调加密密钥 |
| `DEFAULT_USER_ID` | 管理员的企微用户 ID |
| `ADMIN_TOKEN` | 管理员后台密码 |

### 可选

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DEEPSEEK_MODEL` | `deepseek-v3.2` | 模型名 |
| `QWEN_API_KEY` | 空 | Qwen API（不填则全走 DeepSeek） |
| `QWEN_BASE_URL` | 阿里云百炼地址 | Qwen API 地址 |
| `QWEN_MODEL` | `qwen-plus-latest` | Qwen 模型 |
| `QWEN_VL_MODEL` | `qwen-vl-max` | 视觉模型（图片理解） |
| `DAILY_MESSAGE_LIMIT` | `50` | 每人每天消息上限 |
| `WEB_TOKEN_EXPIRE_HOURS` | `24` | Web 令牌有效时长 |
| `WEB_DOMAIN` | `127.0.0.1:9000` | Web 链接域名（**服务器必改**） |
| `DATA_DIR` | `./data` | 数据目录 |
| `INACTIVE_DAYS_THRESHOLD` | `7` | 不活跃天数阈值 |
| `TENCENT_APPID` | 空 | 腾讯云 ASR（语音识别） |
| `TENCENT_SECRET_ID` | 空 | 腾讯云 SecretID |
| `TENCENT_SECRET_KEY` | 空 | 腾讯云 SecretKey |
| `SENIVERSE_KEY` | 空 | 心知天气 API Key |
| `WEATHER_CITY` | `深圳` | 天气查询城市 |

## 成本估算

以 2-3 人日常使用为例：

| 项目 | 月费用 |
|------|--------|
| DeepSeek API | ¥15-50 |
| 服务器（1C1G） | ¥30-60 |
| Qwen/ASR（可选） | ¥0-20 |
| **合计** | **¥45-130/月** |

通过 `DAILY_MESSAGE_LIMIT` + 管理员后台 LLM 用量面板控制成本。
