# KarvisForAll Web 体验审计报告

## 一、审计方法

模拟两位真实用户，从"收到链接"到"日常查看"走完完整用户旅程。
关注点：**这个 Web 端是否让用户觉得"Karvis 真的在帮我管理生活"？**

---

| 维度 | 描述 |
|------|------|

| 技术水平 | 普通用户，日常用微信/抖音/小红书 |

| 核心需求 | 搞清楚"这个东西能帮我干嘛"，看看自己的数据长什么样 |

## 三、用户旅程体验

### 旅程 1：小文的首次体验

#### Step 1：收到链接 → 打开登录页

小文在企微里对 Karvis 说"给我查看链接"，收到一条带 token 的 URL。

**点击链接** → 进入 `/web/login?token=xxx`

✅ **好**：token 从 URL 参数自动填入，无需手动复制粘贴。
✅ **好**：页面设计温暖简洁，🤖 emoji 有辨识度。
⚠️ **困惑**：页面上写着"输入访问令牌"——小文不知道"令牌"是什么。但好在 token 自动填了，她直接点了"进入"。

**结果**：自动跳转到 Dashboard。**顺利**。

#### Step 2：概览页 — "所以……这有什么？"

Dashboard 加载完成。小文看到：

```
Hi, 小文！
2026-02-18 周二

┌──────────┐  ┌──────────┐
│ 📝 今日速记  │  │ ✅ 待办进度  │
│     0     │  │   0/0    │
└──────────┘  └──────────┘
┌──────────┐  ┌──────────┐
│ 🔥 连续打卡  │  │ 📖 最新日记  │
│    0 天   │  │   暂无    │
└──────────┘  └──────────┘
```

**小文的内心**：四个大零。这页面……空空的。

❌ **问题 1：新用户的空状态体验极差**
四个统计卡片全是 0 和"暂无"，没有任何引导告诉用户该做什么。没有"去企微发条消息试试"之类的提示。用户看到全零仪表盘，本能反应是"这东西坏了"或者"没什么用"。

❌ **问题 2：情绪曲线和最近速记区域直接消失**
`x-show` 在无数据时隐藏整个区域，页面只有顶部问候和四个零卡片，底部大片空白。

#### Step 3：点击"速记" Tab

```
📝 速记

── 02 月 16 日 ──

┌───────────────────────┐
│ · 10:45                │
│ https://xxx.trycloudfl │
│ are.com/web/daily      │
├───────────────────────┤
│ · 10:44                │
│ 给我查看链接            │
├───────────────────────┤
│ · 10:42                │
│ 你好                   │
└───────────────────────┘
```

**小文的内心**：？？？

❌ **问题 3（核心痛点）：速记里全是垃圾**
- "你好"是打招呼，不是笔记
- "给我查看链接"是系统指令，不是笔记
- 一段 URL 是 Karvis 的回复（被记成了小文的速记？？），完全不是笔记

3 条"速记"没有一条是用户真正想记录的生活碎片。这直接动摇了"速记"这个功能的可信度——用户会想"它根本不理解什么是有价值的记录"。

#### Step 4：点击其他 Tab

**待办页**：空白，"暂无待办"。
**日记页**：空白，"暂无日记"。
**笔记页**：空白，"暂无笔记"。

✅ 空状态提示文字有，但缺乏行动引导。

#### Step 5：小文想看情绪页面

小文在概览页看到过情绪曲线区域（虽然没数据被隐藏了），想找情绪页面。

底部导航：📊概览 📝速记 ✅待办 📖日记 📁笔记

❌ **问题 4：情绪页面无入口**
底部导航没有"情绪"。情绪页面 `/web/mood` 存在但在标准导航中没有入口。用户永远无法发现它——除非有人给 URL。

#### 小文的体验总结

> "这就是个……啥也没有的空壳？速记里全是我说的废话。感觉这个东西不太聪明的样子。"

**首次体验评分：3/10**

### 旅程 2：空空的日常使用

假设空空已使用一段时间，有真实数据积累。

#### Step 1：概览页 — 还行，但只能看不能动

概览页展示了今日速记数、待办进度、打卡天数、情绪曲线。

✅ **好**：数据一目了然，暖色调舒适。
✅ **好**：情绪曲线用 Chart.js 绘制，视觉效果不错。

⚠️ **问题 5：最近速记预览不可点击**
看到 3 条速记预览，想看完整内容，但点不了——没有跳转。

⚠️ **问题 6：所有内容都是只读**
不能在 Web 端完成待办、新增待办、写笔记。完全的展示型页面。
——这在 MVP 阶段可以接受，但长期会让 Web 端变成"看了就走"的工具，没有粘性。

#### Step 2：速记页 — 信噪比问题

空空翻看速记流。

❌ **问题 7：有价值的内容被噪音淹没**
- "帮我记个待办：明天交报告" → 被记成速记（但已有待办系统处理了）
- "好的" → 被记了
- "做完了" → 被记了
- "今天面试挺顺利" → **这才是真正有价值的速记**

真正的生活碎片被大量指令类消息稀释，翻好几屏才看到一条有意义的内容。

⚠️ **问题 8：速记没有搜索功能**
空空想找"上次说的那个餐厅"，只能一直往下翻。50 条一页，翻了很多页也找不到。

⚠️ **问题 9：速记的 Markdown 渲染非常基础**
只支持链接和换行。如果速记里有代码片段、列表、加粗等内容，全部以纯文本显示。

#### Step 3：日记页 — 还不错，但有 Bug

空空点开日记页，看到日报、周报列表。

✅ **好**：类型标签用颜色区分（日报蓝/周报紫/情绪粉），一目了然。
✅ **好**：点击卡片可展开详情，Markdown 渲染质量可以。

❌ **问题 10：日记页没有"月报" Tab**
筛选 Tab 只有：全部 / 日报 / 周报 / 情绪。但后端会返回 `monthly` 类型。月报在"全部"里可以看到，但无法单独筛选。

❌ **问题 11（BUG）：情绪日记点击 404**
日记列表中情绪日记的 `file` 值为 `emotion/xxx.md`，但详情接口使用 `os.path.basename()` 截断了 `emotion/` 前缀，在 `daily_notes_dir` 下找不到该文件。**用户点击情绪日记条目 → 404 白屏**。

⚠️ **问题 12：详情加载无反馈**
点击卡片后，HTTP 请求需要几百毫秒。这期间没有 loading 态——页面看起来"卡住了"。

#### Step 4：笔记/归档页 — 功能完整但朴素

✅ **好**：6 个分类清晰（工作/情感/生活/读书/影视/语音），可横滑。
✅ **好**：列表→详情→返回，交互自然。

⚠️ **问题 13：分类按钮太小**
`px-2 py-1` 在移动端触控面积偏小，容易误触相邻按钮。

#### Step 5：空空想看"Karvis 记住了我什么"

空空和 Karvis 聊了很多，包括自己的职业、朋友、偏好。他想看看 Karvis 到底记住了哪些。

**打开概览** → 没有。
**翻遍所有 Tab** → 没有。

❌ **问题 14（核心缺失）：完全无法查看 AI 的记忆**
memory.md 是 Karvis 对用户的认知画像（用户画像/重要的人/偏好/近期关注/重要事件），直接影响 AI 的所有行为。但用户：
- 看不到 Karvis 记住了什么
- 无法确认记忆是否准确
- 无法主动校正错误记忆

这对"数字生活管家"来说是致命缺失——管家应该让主人知道"我对您的了解是这些"。

#### Step 6：空空想看情绪趋势

概览页有 7 天情绪曲线，但空空想看更长周期和详细情绪日记。

❌ **问题 15（导航死胡同）：概览页的情绪曲线不可点击**
看到了曲线，但无法点击跳转到详情。而底部导航里没有"情绪"入口。

实际上 `/web/mood` 页面存在且功能完整（30 天曲线+统计+日记列表），但用户完全无法发现它。

#### 空空的体验总结

> "基本能用，信息展示还行。但速记里太多垃圾了，每次翻半天。最大的遗憾是看不到 Karvis 对我的记忆——我不知道它到底了解我多少。情绪页面居然找不到入口？另外每个页面都只能看不能操作，有点鸡肋。"

**日常使用评分：6/10**

## 五、优化建议

### 第一优先：解决"这是什么"和"有什么用"

> 用户打开 Web 端的第一感受决定了他会不会再来。

**1. 速记智能过滤（解决 E-1）**

当前所有消息无差别写入 Quick-Notes。应基于 LLM 的 skill 决策结果过滤：
- `todo.add/done/list` → 跳过（待办系统已处理）
- `web.token` / `settings.*` → 跳过（系统指令）
- `checkin.*` → 跳过（打卡系统已处理）
- `ignore` + 短文本正则匹配（"你好""嗯""好的"） → 跳过

零额外 LLM 调用，纯代码逻辑。

**2. 新增记忆页面（解决 E-2）**

新增 `/api/memory` 端点 + `memory.html` 页面：
- 解析 memory.md 的 `## 章节` → 按分区展示
- 入口放在 Dashboard 概览页顶部（用户画像入口卡片）
- 底部提示"有错的地方在企微告诉我就好~"，形成校正闭环

**3. 修复导航（解决 E-3 + U-1）**

统一所有页面的底部导航为 5 个：**📊概览 📝速记 ✅待办 📖日记 📁笔记**

情绪页面入口方案：
- Dashboard 情绪曲线区域可点击 → 跳转 `/web/mood`
- 日记页的情绪 Tab 筛选条目可点击 → 跳转 `/web/mood`

不在底部导航占位（情绪是低频查看场景）。

**4. 空状态引导（解决 E-4）**

新用户首次打开 Dashboard 时，用引导卡片替代空数据：

```
👋 欢迎来到你的数字空间！

在企微里对 Karvis 说点什么，就能开始记录生活：

💬 "今天面试挺顺利的" → 自动记录速记
📝 "帮我记个待办：交报告" → 创建待办
📖 "刚看完三体太震撼了" → 影视笔记

去试试吧~
```

### 第二优先：修复 Bug + 体验一致性

**5. 修复情绪日记 404（解决 B-1）**

`web_routes.py` 的 `/api/daily/<filename>` 端点中，如果 filename 包含 `emotion/` 前缀，应去 `emotion_notes_dir` 而非 `daily_notes_dir` 查找。

**6. 统一技术栈（解决 U-2 + D-1）**

将 `notes.html` 改造为 Alpine.js + Tailwind CSS + fetch，与其他页面保持一致。保留入场动画效果。统一背景色为 `#fdf8f6`。

**7. 日记页补"月报" Tab（解决 U-3）**

在筛选 Tab 中增加"月报"选项。

### 第三优先：增值功能

**8. 概览页速记可点击（解决 U-5）**

最近速记卡片点击跳转到 `/web/notes`。

**9. Dashboard 情绪曲线可点击（辅助解决 E-3）**

情绪卡片点击跳转到 `/web/mood`。

**10. 详情加载态（解决 U-6）**

日记/笔记点击后显示骨架屏或 loading spinner。

### 远期方向

| 方向 | 说明 |
|------|------|
| 搜索能力 | 全局搜索速记/笔记/日记内容 |
| 写入能力 | Web 端完成待办、新增速记 |
| PWA 支持 | 添加到主屏幕、离线访问 |

## 六、核心反思

当前 Web 端的根本问题不是"页面不好看"，而是 **没有回答"这个 Web 端为什么要存在"这个问题**。

8 个页面中，7 个是数据展示、1 个是登录。所有内容都是只读的"历史记录查看器"。它缺少两个让用户愿意回来的东西：

1. **认知可见性**——"Karvis 了解我多少"
   记忆页面不只是新增一个功能，它是让用户感受到"这个 AI 真的在关注我"的可视化呈现。

2. **内容质量感**——"我的数字生活值得回看"
   速记应该是精选的生活碎片时间线，而不是企微聊天记录的复制品。当用户打开速记页看到的全是"你好""嗯""ok"，他不会觉得 Karvis 在帮他管理生活。

这两点改好了，Web 端才真正服务于"个人数字生活管家"这个定位。
