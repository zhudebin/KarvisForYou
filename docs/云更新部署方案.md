# KarvisForAll — 云更新部署方案

## 架构概览

```
┌─────────────────┐     git push deploy master     ┌──────────────────────────┐
│  本地开发机 Mac   │ ────────────────────────────►  │  腾讯云 Lighthouse        │
│                  │         SSH (需微信扫码)         │  119.29.237.199          │
│  ~/01_Obsidian/  │                                 │                          │
│  _TOOLs/         │                                 │  bare repo:              │
│  KarvisForAll/   │                                 │    ~/karvis-repo.git     │
│                  │                                 │                          │
│  branch: master  │                                 │  work tree:              │
│  remote: deploy  │                                 │    ~/KarvisForAll/       │
└─────────────────┘                                  │                          │
                                                     │  进程: python3 app.py    │
                                                     │  端口: 9000              │
                                                     │  日志: /root/karvis.log  │
                                                     └──────────────────────────┘
```

**部署方式**：Git Push → post-receive 钩子自动部署（非 Docker，直接进程）

---

## 一、日常更新流程

### 1. 提交并部署（一行命令）

```bash
cd KarvisForAll
git add . && git commit -m "feat: 你的改动描述"
git push deploy master
```

push 完成后，服务器 `post-receive` 钩子自动执行：

```
拉取最新代码 → 安装/更新依赖 → kill 旧进程 → 启动新进程
```

### 2. 验证部署成功

push 输出中会看到：

```
remote: >>> 开始部署 KarvisForAll...
remote: >>> 部署完成！PID: xxxxx
```

也可手动确认：

```bash
ssh root@119.29.237.199 'pgrep -f "python3 app.py" && echo "✅ 运行中" || echo "❌ 未运行"'
```

### 3. 查看日志

```bash
ssh root@119.29.237.199 'tail -50 /root/karvis.log'        # 最近50行
ssh root@119.29.237.199 'tail -f /root/karvis.log'          # 实时跟踪
ssh root@119.29.237.199 'grep -i error /root/karvis.log | tail -20'  # 搜错误
```

## 二、服务器端配置详情

### post-receive 钩子

位置：`~/karvis-repo.git/hooks/post-receive`

```bash
#!/bin/bash
echo ">>> 开始部署 KarvisForAll..."
WORK_DIR=/root/KarvisForAll

# 拉取最新代码
git --work-tree=$WORK_DIR --git-dir=/root/karvis-repo.git checkout -f

# 安装依赖
cd $WORK_DIR/src
pip3 install -r requirements.txt --quiet

# 重启服务
PID=$(pgrep -f "python3 app.py" | head -1)
if [ -n "$PID" ]; then
    kill $PID
    sleep 2
fi
cd $WORK_DIR/src
nohup python3 app.py > /root/karvis.log 2>&1 &
NEW_PID=$!
echo ">>> 部署完成！PID: $NEW_PID"
```

### 目录结构

```
~/karvis-repo.git/           # bare repo（接收 git push）
    hooks/post-receive       # 自动部署钩子

~/KarvisForAll/              # 工作目录（运行代码）
    src/
        app.py               # 入口，Flask，端口 9000
        .env                 # 环境变量（不入 Git）
        requirements.txt
    data/
        _karvis_system/      # 系统数据
            users.json       # 用户注册表
            tokens.json      # Web 登录令牌
            usage_log.jsonl  # 用量日志
        users/
            <user_id>/       # 每个用户的独立数据目录

/root/karvis.log             # 应用日志
```

### 本地 Git 配置

```ini
# .git/config
[remote "deploy"]
    url = root@119.29.237.199:~/karvis-repo.git
    fetch = +refs/heads/*:refs/remotes/deploy/*
```

## 三、常用运维操作

### 手动重启

```bash
ssh root@119.29.237.199 'kill $(pgrep -f "python3 app.py"); sleep 1; cd ~/KarvisForAll/src && nohup python3 app.py > /root/karvis.log 2>&1 &'
```

### 清空用户数据（重置）

```bash
ssh root@119.29.237.199 'rm -rf ~/KarvisForAll/data/users/*'
ssh root@119.29.237.199 'echo "{\"users\":{}}" > ~/KarvisForAll/data/_karvis_system/users.json'
ssh root@119.29.237.199 'echo "{}" > ~/KarvisForAll/data/_karvis_system/tokens.json'
ssh root@119.29.237.199 'truncate -s 0 ~/KarvisForAll/data/_karvis_system/usage_log.jsonl'
```

清空后需重启进程。

### 备份数据

```bash
ssh root@119.29.237.199 'cd ~/KarvisForAll && tar czf /root/karvis-backup-$(date +%Y%m%d).tar.gz data/'
```

### 恢复数据

```bash
scp root@119.29.237.199:/root/karvis-backup-20260218.tar.gz .
ssh root@119.29.237.199 'cd ~/KarvisForAll && tar xzf /root/karvis-backup-20260218.tar.gz'
```

### 手动触发定时任务

```bash
ssh root@119.29.237.199 'curl -sX POST http://localhost:9000/system -H "Content-Type: application/json" -d "{\"action\":\"morning_report\"}"'
```

支持的 action：`morning_report` / `daily_report` / `refresh_cache` / `mood_generate` / `weekly_review` / `monthly_review` / `nudge_check` / `companion_check` / `todo_remind` / `evening_checkin`

## 四、SSH 注意事项

- 腾讯云 Lighthouse 的 SSH 安全策略要求**微信扫码认证**，每次 SSH 连接都会触发
- 复杂命令建议拆成多条 SSH 执行，避免 `nohup ... &` 与 `&&` 组合的语法问题
- 建议在命令中避免单引号嵌套，优先使用双引号 + 转义

## 五、故障排查

### push 失败

| 问题 | 原因 | 解决 |
|------|------|------|
| `源引用规格 main 没有匹配` | 分支名不对 | 用 `master` 不是 `main` |
| `Permission denied` | SSH 密钥问题 | 检查 `~/.ssh/` 配置 |
| 微信扫码超时 | 腾讯云安全策略 | 重新 push，及时扫码 |

### 部署后服务异常

1. 检查进程：`pgrep -f "python3 app.py"`
2. 查日志：`tail -100 /root/karvis.log`
3. 检查端口：`ss -tlnp | grep 9000`
4. 检查 .env：`cat ~/KarvisForAll/src/.env | head -5`（确认环境变量存在）

### 常见日志关键词

| 关键词 | 含义 |
|--------|------|
| `[handle_message]` | 收到企微消息 |
| `[Brain]` | AI 处理逻辑 |
| `reply_text` | 回复发送 |
| `ERROR` / `Traceback` | 程序异常 |
| `flash_filter` | 速记 Flash 过滤 |

## 六、与个人版 Karvis 的区别

| | Karvis（个人版） | KarvisForAll（多用户版） |
|---|---|---|
| 服务器 | 同一台 119.29.237.199 | 同一台 |
| 部署目录 | `/opt/karvis` | `~/KarvisForAll` |
| 部署方式 | Docker Compose | Git Push + 直接进程 |
| 端口 | 9001 | 9000 |
| push 命令 | `git push prod main` | `git push deploy master` |
| 日志 | `docker compose logs` | `/root/karvis.log` |
| zshrc 快捷命令 | `karvis-deploy` 等（已配置） | 暂无 |

> 两套服务在同一台机器上共存，互不干扰。
