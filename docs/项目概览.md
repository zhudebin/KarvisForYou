# KarvisForAll — 项目概览

## 项目信息

| 字段 | 值 |
|------|-----|
| 名称 | KarvisForAll |
| 一句话介绍 | 基于企微的多用户 AI 生活助手，2-5 人共享部署，每人独立数据空间，多模型调度（DeepSeek V3.2 + Qwen Flash） |
| 本地源码路径 | `_TOOLs/KarvisForAll/` |
| 与 Karvis 的关系 | 从 Karvis（个人版）fork 后进行多用户改造的独立项目 |
| 部署平台 | 腾讯云 Lighthouse 云服务器 / 任意 Linux 服务器 / 本地电脑 |
| 运行环境 | Python 3.9+, Flask, Docker(可选), APScheduler 内嵌 |
| 部署方式 | 三选一：一键脚本 `setup.sh` / Docker Compose / 手动 `python app.py` |
| 云服务器部署 | `git push deploy master` → post-receive 钩子自动拉代码 → 安装依赖 → 重启进程 |
| 存储 | 服务器本地磁盘（Lite 模式），不使用 OneDrive |
| LLM | 四层模型架构：Flash(Qwen Flash/阿里云) + Main(DeepSeek V3.2 thinking=off) + Think(DeepSeek V3.2 thinking=on) + VL(Qwen VL Max/阿里云) |

## 多用户核心设计

### UserContext — 全链路贯穿

```
企微消息 → app.py 解密得到 user_id
  → get_or_create_user(user_id) 得到 UserContext
  → brain.process(payload, ctx, send_fn)
    → read_state_cached(ctx)         # 读该用户的 state
    → load_memory(ctx)               # 读该用户的 memory
    → handler(params, state, ctx)    # skill 使用 ctx 的路径
  → send_wework_message(user_id, reply)
```

每个用户的 `UserContext` 包含该用户所有文件路径（`inbox_path`、`quick_notes_file`、`state_file`、`todo_file`、`memory_file` 等），所有数据操作通过 `ctx` 路径访问，确保用户间数据完全隔离。

### 数据隔离保障

| 层级 | 隔离机制 |
|------|---------|
| 文件系统 | `users/{user_id}/` 独立目录，UserContext 路径前缀校验 |
| 缓存 | State 缓存按 `user_id` 分区：`_state_cache[user_id]` |
| 定时任务 | 遍历活跃用户逐一执行，per-user 上下文 |
| Web 访问 | Token → user_id 映射，只能查看自己的数据 |

## 与 Karvis（个人版）的差异

| 维度 | Karvis | KarvisForAll |
|------|--------|-------------|
| 用户模式 | 单用户，全局 state | 多用户，per-user UserContext |
| 存储 | OneDrive (Graph API) | 本地磁盘 (LocalFileIO) |
| 数据查看 | Obsidian 直接编辑 | Web 只读页面 + Token 鉴权 |
| 财务管理 | 有（4 个 finance.* skill） | 无（个人敏感功能不适合多用户） |
| 用户管理 | 无 | 自动注册 + 欢迎引导 + 管理员后台 |
| 成本控制 | 无 | 每日消息上限 + LLM 用量追踪 |
| 部署 | git push prod main | 一键脚本 / Docker / 手动 / git push deploy |
| handler 签名 | `(params, state)` | `(params, state, ctx)` |
| 新增 Skill | settings.* / web.token | — |

## 性能关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `STATE_CACHE_TTL` | 300s (5分) | State 内存缓存，按 user_id 分区 |
| `PROMPT_CACHE_TTL` | 1800s (30分) | Prompt 文件缓存 |
| `RECENT_MESSAGES_LIMIT` | 10 条 | 短期记忆滑动窗口 |
| `MSG_CACHE_EXPIRE_SECONDS` | 60s | 消息去重窗口 |
| `CHECKIN_TIMEOUT_SECONDS` | 43200s (12h) | 打卡超时 |
| `SCHEDULER_TICK_MINUTES` | 30 | V8 智能调度心跳间隔 |
| `SCHEDULER_PUSH_MAX_DAILY` | 6 | 每日主动推送上限 |
| `SCHEDULER_MIN_PUSH_GAP` | 30 分钟 | 两次推送最小间隔 |
| `SCHEDULER_RHYTHM_WINDOW` | 7 天 | 节奏学习滑动窗口 |
| `DAILY_MESSAGE_LIMIT` | 50 | 每用户每日消息上限 |
| `WEB_TOKEN_EXPIRE_HOURS` | 24h | Web 查看令牌有效期 |
| `SERVER_PORT` | 9000 | Flask 服务端口 |
